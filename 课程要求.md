# 开发任务说明：北斗定位解算全流程软件系统

> 用途：把课程 PDF 题目整理为 编程 Agent 易读的工程需求文档。  
> 建议文件名：`AGENTS.md` 或 `requirements.md`。如果放进仓库根目录，可作为项目级开发规范和任务清单。

---

## 1. 项目目标

开发一个纯软件的北斗定位解算全流程系统。系统基于广播星历导航文件 `*.nav`，通过给定的伪距误差生成模型模拟伪距观测值，完成：

1. 导航数据解析与伪距观测值模拟；
2. 北斗卫星位置与钟差计算；
3. 伪距单点定位（SPP）解算；
4. 多历元连续定位、精度分析与可视化；
5. 模块化系统整合、GUI 交互与测试报告输出；
6. 可选：基于机器学习的定位误差预测与补偿。

核心算法必须手写实现，禁止直接调用第三方定位库完成卫星位置解算、最小二乘定位等核心功能。

优先实现 Python 版本，便于课程验收、可视化和 GUI 开发。

允许使用：

- Python 3.10+
- `numpy`：矩阵、最小二乘、坐标计算
- `pandas`：结果表格保存
- `matplotlib`：误差曲线、轨迹图、DOP 曲线

- RINEX 解析、卫星位置、钟差、传播延迟、DOP、迭代最小二乘等核心模块需要自己实现。
- 不要把完整代码或报告与同学共享；课程会做大模型/代码雷同检测。

## 4. 功能模块需求

## Module 1: 导航数据解析与伪距观测值模拟

### 目标

解析 RINEX 导航文件，提取定位所需星历数据；并基于给定的伪距误差生成模型模拟各历元伪距观测值。

### 必须实现

1. 解析导航文件 `*.nav`：
   - 卫星编号；
   - 星历参考时间；
   - 卫星钟差参数；
   - 广播星历轨道参数；
   - 卫星健康状态。

2. 伪距观测值模拟：
   - 基于卫星真实位置与接收机概略位置计算几何距离 `rho`；
   - 采用北斗三号标准伪距误差生成模型模拟伪距观测值：
   
   ```python
   # 北斗三号标准伪距（完全官方量级）
   P = rho + random.gauss(0, 0.5)       # SISRE
         + random.gauss(10, 3)           # 电离层
         + random.gauss(4, 1.5)          # 对流层
         + (60 + random.gauss(0, 12))    # 接收机钟差
         + random.gauss(0, 0.3)          # 噪声
   ```
   
   - 按历元生成多卫星模拟观测数据；
   - 可配置随机种子以保证结果可复现。

3. 数据预处理：
   - 剔除不健康卫星；
   - 伪距粗差初步剔除；
   - 按高度角筛选可见卫星；
   - 实现同一历元多卫星数据时间对齐。

### 建议接口

```python
class RinexNavParser:
    def parse(self, nav_path: str) -> dict[str, list[BroadcastEphemeris]]:
        ...

def simulate_pseudorange(
    rho_m: float,
    use_iono: bool = True,
    use_trop: bool = True,
    use_clock_bias: bool = True,
    random_seed: int | None = None,
) -> float:
    ...

def preprocess_epoch(
    epoch: SimulatedEpoch,
    ephemerides: dict[str, list[BroadcastEphemeris]],
    elevation_mask_deg: float,
) -> ProcessedEpoch:
    ...
```

## Module 2: 卫星位置、钟差与传播延迟修正

### 目标

基于广播星历计算北斗卫星在 ECEF 坐标系下的位置，并完成卫星钟差、对流层、电离层等修正。

### 必须实现

1. 卫星位置计算：
   - 根据广播星历计算卫星 ECEF 坐标；
   - 包含轨道摄动修正；
   - 输出卫星 `X/Y/Z` 坐标。

2. 卫星钟差修正：
   - 多项式钟差修正；
   - 相对论效应修正；
   - 用钟差修正伪距观测值。

3. 传播延迟修正：
   - 使用 Saastamoinen 模型完成对流层延迟修正；
   - 使用简化模型完成电离层延迟修正；
   - 输出最终修正后的伪距。

### 建议接口

```python
def compute_satellite_position(
    eph: BroadcastEphemeris,
    transmit_time: float,
) -> EcefCoordinate:
    ...

def compute_satellite_clock_bias(
    eph: BroadcastEphemeris,
    transmit_time: float,
) -> float:
    ...

def correct_pseudorange(
    raw_pseudorange_m: float,
    sat_clock_bias_s: float,
    trop_delay_m: float,
    iono_delay_m: float,
) -> float:
    ...
```

### 验收要点

- 卫星位置单位为米；
- 钟差单位统一，秒和米之间转换清晰；
- 所有关键公式应有注释，说明变量含义；
- 输出中间调试文件，便于验收。

## Module 3: 伪距单点定位 SPP 核心算法

### 目标

构建伪距观测方程，采用迭代最小二乘算法求解接收机 ECEF 坐标与接收机钟差，并转换为经纬高结果。

### 必须实现

1. 可见卫星筛选：
   - 每个历元至少需要 4 颗可用卫星；
   - 计算并输出可用卫星数量；
   - 计算 `PDOP`、`GDOP` 等几何精度因子。

2. 迭代最小二乘定位：
   - 构建伪距观测方程；
   - 未知量：`x, y, z, receiver_clock_bias_m`；
   - 设置最大迭代次数；
   - 设置收敛阈值；
   - 输出每个历元的位置解和钟差。

3. 坐标转换：
   - ECEF 转 BLH；
   - 输出经度、纬度、高程。

### 建议接口

```python
def solve_spp_epoch(
    processed_epoch: ProcessedEpoch,
    initial_position: EcefCoordinate | None,
    max_iter: int,
    convergence_threshold_m: float,
) -> PositionSolution:
    ...

def compute_dop(geometry_matrix: np.ndarray) -> DopResult:
    ...

def ecef_to_blh(x: float, y: float, z: float) -> BlhCoordinate:
    ...
```

### 验收要点

- 至少 4 颗卫星时才能解算；
- 迭代过程需要判断是否收敛；
- 解算失败时不要崩溃，应记录失败原因；
- 每个历元保存定位结果与 DOP。

class SimulatedObservation:

    pseudorange_m: float

    epoch: datetime

    sat_id: str

    toc: float
    clock_bias: float

    clock_drift_rate: float

    sat_id: str
    x_m: float
    y_m: float
    z_m: float
    clock_bias_s: float
    elevation_deg: float
    corrected_pseudorange_m: float

class PositionSolution:
    epoch: datetime
    x_m: float
    y_m: float
    z_m: float
    lat_deg: float
    lon_deg: float
    height_m: float
    receiver_clock_bias_m: float
    num_sats: int
    gdop: float
    pdop: float

## 14. Done Definition

当满足以下条件时，可认为项目完成：

- [ ] 能从 GUI 或 CLI 导入 RINEX 导航文件；
- [ ] 能基于伪距误差生成模型模拟多历元、多卫星伪距观测值；
- [ ] 能计算卫星位置与钟差；
- [ ] 能完成传播延迟修正；
- [ ] 能进行单历元 SPP 定位；
- [ ] 能进行多历元连续定位；
- [ ] 能输出经纬高定位结果；
- [ ] 能输出 DOP、卫星数量和精度指标；
- [ ] 能生成至少 1 张可视化图；
- [ ] GUI 可用；
- [ ] 多组数据测试通过；
- [ ] 设计报告、实验报告、测试报告完整；
- [ ] 核心算法为手写实现，没有直接调用第三方定位库。
