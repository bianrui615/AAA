# 北斗定位解算全流程软件系统 — 实验报告

## 1. 实验目的

本实验实现一套基于 BDS-3 CNAV 广播星历的单点定位（SPP）流程，并在基础定位结果上加入机器学习误差补偿。当前项目不读取真实 OBS 文件，而是通过广播星历、真实接收机坐标和模拟伪距误差模型完成可复现实验。

实验目标包括：

1. 解析 RINEX 3.x CNAV 导航文件。
2. 计算健康北斗卫星的 ECEF 坐标和卫星钟差。
3. 生成含 SISRE、电离层、对流层、接收机钟差和噪声的模拟伪距。
4. 使用迭代最小二乘完成单历元和连续历元 SPP 解算。
5. 输出误差统计、DOP 分析和轨迹/误差图表。
6. 通过 PyQt5 GUI 完成参数配置、运行控制和结果展示。
7. 使用线性回归与随机森林对三维坐标误差进行预测和补偿。

## 2. 实验环境

主要依赖如下：

```text
numpy
matplotlib
PyQt5
scikit-learn
joblib
```

导航文件放在 `nav/` 目录下，当前项目包含 `tarc0910.26b_cnav` 至 `tarc1300.26b_cnav` 等 BDS-3 CNAV 文件。

## 3. 实验流程

基础部分可通过命令运行：

```powershell
python basic/module5.py
```

也可通过 GUI 运行：

```powershell
python basic/rinex_gui.py
```

三场景批量评估入口为：

```powershell
python basic/gui_scenario_runner.py
```

提高部分入口为：

```powershell
python enhance/run_enhance.py
```

| 模型 | 补偿前 RMS | 补偿后 RMS | RMS 改善 |

| 线性回归 | 2.055674 m | 2.161443 m | -5.15% |

## 7. 结论

项目当前已完成基础定位流程、单 NAV GUI、三场景批量 GUI、提高部分命令行流程和提高部分 GUI。基础部分在默认数据上可稳定输出米级模拟定位误差；提高部分完成了数据集构建、模型训练、坐标补偿和报告生成，但现有小规模三场景数据上的补偿效果仍需进一步优化。
