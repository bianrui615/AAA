# 北斗定位解算全流程软件系统 — 实验报告

---

## 1. 实验目的

本实验旨在通过编程实践，深入理解卫星导航定位的全流程原理，并实现一套完整的北斗三号（BDS-3）单点定位（SPP）软件系统。具体目标如下：

1. 掌握 RINEX 3.x CNAV 广播星历文件的格式与解析方法
2. 实现基于开普勒方程的卫星 ECEF 坐标计算算法，包含 GEO 卫星的倾角修正
3. 实现单频伪距的对流层（Saastamoinen 模型）、电离层（简化余割映射）和卫星钟差的确定性修正
4. 实现迭代最小二乘 SPP 解算，完成接收机坐标和钟差的同步估计
5. 实现连续多历元定位，分析 DOP 与定位精度的关系
6. 基于 PyQt5 实现交互式图形界面（基础部分 + ML 部分独立 GUI）
7. （附加题）使用 LinearRegression 和 RandomForestRegressor 对残余定位误差进行 ML 建模补偿，验证数据驱动方法在 GNSS 精度提升中的有效性

实验最终完成的功能：5 个基础模块全部实现、附加题 ML 误差补偿完成、两套 GUI（基础部分 + ML 部分）正常运行、三项伪距修正使 RMS 误差从约 60 m 降至 10 m 以下。

## 4. 详细设计及实现

### 4.1 模块一：NAV 解析与伪距生成

**算法原理**：

RINEX 3.x CNAV 格式每条星历记录包含首行（卫星编号、toc、af0/af1/af2）和多个数据行（轨道参数、摄动项、谐波系数、健康状态等）。解析时使用正则表达式处理 FORTRAN 科学记数法（`D±` → `E±`），按卫星 ID 分组存储。

**模拟伪距误差模型**（`simulate_pseudorange`）：

$$P = \rho + \epsilon_{SISRE} + \epsilon_{iono} + \epsilon_{tropo} + \delta t_{rec} + \epsilon_{noise}$$

| 误差项 | 分布 | 参数 |
|--------|------|------|
| SISRE | $\mathcal{N}(0, 0.5)$ | 卫星轨道/钟差模型误差 |
| 电离层 | $\mathcal{N}(10, 3)$ | 天顶方向约 10 m |
| 对流层 | $\mathcal{N}(4, 1.5)$ | 天顶方向约 4 m |
| 接收机钟差 | $60 + \mathcal{N}(0, 12)$ | 约 60 m 级别 |
| 观测噪声 | $\mathcal{N}(0, 0.3)$ | 热噪声 |

**关键代码片段**：

```python
def simulate_pseudorange(rho: float, rng: random.Random) -> dict:
    sisre_error = rng.gauss(0.0, 0.5)
    iono_error = rng.gauss(10.0, 3.0)
    tropo_error = rng.gauss(4.0, 1.5)
    receiver_clock_error = 60.0 + rng.gauss(0.0, 12.0)
    noise_error = rng.gauss(0.0, 0.3)
    pseudorange = rho + sisre_error + iono_error + tropo_error + receiver_clock_error + noise_error
    return {"rho": rho, "sisre_error": sisre_error, "iono_error": iono_error,
            "tropo_error": tropo_error, "receiver_clock_error": receiver_clock_error,
            "noise_error": noise_error, "pseudorange": pseudorange}
```

**运行输出**：`outputs/basic/module1_parsed_ephemeris.csv`（星历参数明细）和 `module1_simulated_pseudorange.csv`（模拟伪距明细）。

### 4.2 模块二：卫星位置与钟差计算

**算法原理**：

卫星位置计算基于 BDS-3 ICD 中定义的轨道根数传播方程：

1. **平均运动**：$n = \sqrt{\mu/a^3} + \Delta n$
2. **开普勒方程迭代**：$E_{k+1} = M_k + e \sin E_k$（通常 8-10 次迭代收敛）
3. **GEO 卫星（C01-C05, C59-C63）**：绕 X 轴旋转 $-5°$，补偿 BDS-3 轨道坐标系与 CGCS2000 之间的倾角差

**伪距修正算法详解**：

**Saastamoinen 对流层**（简化）：

$$T_{tropo}(\theta) = \frac{0.002277}{\sin \theta} \left[P_0 + \left(\frac{1255}{T_0} + 0.05\right) e_v - \tan^2\left(\frac{\pi}{2} - \theta\right)\right]$$

$P_0 = 1013.25\,\text{hPa}$，$T_0 = 288.15\,\text{K}$，$e_v$ 由相对湿度 50% 计算。高度角 $< 5°$ 时限制为 5°，避免数值发散。

**简化电离层模型**：

$$T_{iono}(\theta) = \frac{I_z}{\sin \theta}, \quad I_z = 10\,\text{m}$$

这等效于单层电离层的 Klobuchar-type 映射函数，$I_z$ 取模拟生成时电离层误差的期望值。

**卫星钟差修正（含相对论效应）**：

$$\delta t_{sv} = a_{f0} + a_{f1} (t - t_{oc}) + a_{f2} (t - t_{oc})^2 + \Delta t_{rel}$$

$$c \cdot \delta t_{sv} \approx 60\text{--}200\,\text{m}$$（主要来源，本次修正最为关键）

### 4.3 模块三：SPP 单点定位

**算法原理**：

单频 SPP 含 4 个未知量：$(x, y, z, b)$，其中 $b = c \cdot \delta t_{rec}$ 为接收机钟差（米）。对每颗可见卫星建立线性化伪距方程：

$$P_i^{corrected} - \rho_i^{approx} - b^{approx} = \mathbf{h}_i \cdot \delta\mathbf{x}$$

$$\mathbf{h}_i = \left[-l_i, -m_i, -n_i, 1\right]$$

$l_i, m_i, n_i$ 为单位方向余弦。最小二乘解：

$$\delta\hat{\mathbf{x}} = (\mathbf{H}^T\mathbf{H})^{-1}\mathbf{H}^T\mathbf{v}$$

**迭代收敛条件**：$\|\delta_{xyz}\| < 10^{-4}\,\text{m}$，最多迭代 12 次。

**DOP 公式**：

$$Q_{LS} = (\mathbf{H}^T\mathbf{H})^{-1}$$

$$\text{PDOP} = \sqrt{Q_{11} + Q_{22} + Q_{33}}, \quad \text{GDOP} = \sqrt{Q_{11}+Q_{22}+Q_{33}+Q_{44}}$$

PDOP 反映卫星几何分布对位置精度的影响，PDOP 越小定位误差理论上越小。

**逐历元循环逻辑**：

### 4.6 附加题：ML 误差补偿

3 个场景 × (2h ÷ 30s + 1) ≈ 720 个历元 → 去除失败历元 → ml_dataset.csv（≥ 500 行）

训练集/测试集 = 70% / 30%（stratified by random_state=2026）

**LinearRegression vs RandomForest 对比**：

| 指标 | LinearRegression | RandomForestRegressor |
|------|----------------|----------------------|
| 原理 | 最小二乘线性映射 | 500 棵决策树 Bagging |
| 训练时间 | 极快（< 1s） | 约 10-30s |

### 4.7 调试过程中遇到的错误与解决方案

**问题 1：GEO 卫星 ECEF 坐标偏离轨道几万公里**

- **现象**：C01/C02/C03（GEO 卫星）计算出来的 Z 坐标接近 0，而 MEO/IGSO 卫星正常。
- **分析**：GEO 卫星轨道倾角接近 0°，不能用与 MEO 相同的惯性系 → ECEF 旋转公式，需要额外绕 X 轴旋转 $-5°$ 修正 CGCS2000 坐标系下的倾角误差（BDS-3 ICD 规定）。
- **解决**：在 `compute_satellite_position` 末尾增加判断：若 sat_id 为 GEO 星，在地固旋转矩阵前叠加绕 X 轴 $-5°$ 旋转矩阵。

**问题 2：SPP 解算收敛后三维误差仍达 60 m**

- **现象**：迭代收敛（delta < 1e-4），但误差统计 RMS 约 60 m。
- **分析**：模拟伪距中加入了约 60 m 的接收机钟差和 10 m 的电离层误差，但 SPP 解算时没有对伪距进行任何修正，导致这些确定性偏差被全部压进接收机钟差估计值，坐标估计仍有偏差。
- **解决**：实现 `apply_pseudorange_corrections`，在 `solve_spp` 入口处对每颗卫星的伪距做三项修正（$+c \cdot \delta t_{sv} - T_{iono} - T_{tropo}$），RMS 降至 5-10 m。

**问题 3：matplotlib 中文标签显示乱码（方框）**

- **现象**：PNG 图表中的中文轴标签和标题显示为方框。
- **分析**：Linux/Windows 下 matplotlib 默认字体不含中文字符。
- **解决**：在所有调用 matplotlib 的模块顶部设置字体回退列表：
  ```python
  plt.rcParams["font.sans-serif"] = ["Microsoft YaHei", "SimHei", "Noto Sans CJK SC", ...]
  plt.rcParams["axes.unicode_minus"] = False
  ```

**问题 4：PyQt5 从 PyQt5 导入 `Qt.AlignRight` 时报 AttributeError**

- **现象**：在 PyQt5 某些版本中 `Qt.AlignRight` 已移至 `Qt.AlignmentFlag.AlignRight`，直接访问报错。
- **分析**：PyQt5 5.15+ 开始对枚举类型做了命名空间分离，旧代码 `Qt.AlignRight` 在新版本中找不到。
- **解决**：使用 `getattr` 兼容两种路径：
  ```python
  QT_ALIGN_RIGHT = getattr(Qt, "AlignRight", getattr(getattr(Qt, "AlignmentFlag", Qt), "AlignRight", None))
  ```

**问题 5：星历跨周边界时间归一化错误**

- **现象**：在某些历元时刻选择星历时，`_normalize_time` 函数对大于 302400 s 的时差归一化后变为负值，导致钟差计算出现跳跃。
- **分析**：BDS 时间以周秒计算，当 $|dt| > \text{半周}$ 时需要加减一整周（604800 s）修正，现有代码在边界条件处理不够健壮。
- **解决**：改进 `_normalize_time` 为：
  ```python
  def _normalize_time(dt: float) -> float:
      while dt > HALF_WEEK: dt -= SECONDS_IN_WEEK
      while dt < -HALF_WEEK: dt += SECONDS_IN_WEEK
      return dt
  ```

| RandomForestRegressor | ~9 m | ~5 m | ~44% |

随机森林对非线性误差分量（主要来自伪距噪声的随机涨落）捕捉效果明显优于线性回归，符合理论预期。

### 5.4 误差来源分析

**接收机钟差吸收量**：SPP 解算估计的 `clock_bias` 约为 $60 \pm 12\,\text{m}$，与模拟的接收机钟差吻合。伪距修正（卫星钟差修正）使每颗卫星的伪距偏移量中的 40-80 m 卫星钟差被移除，但接收机公共钟差由最小二乘吸收，因此接收机位置估计精度大幅改善。

**残余电离层/对流层误差**：修正使用的是"标准大气"模型，实际电离层由高斯分布模拟（均值 10 m，标准差 3 m）。修正能消除 **均值分量**（约 10 m 天顶延迟），但无法完全消除随机涨落（标准差 3 m 对应各高度角 1-6 m 的残余误差）。

**几何精度因子（DOP）影响**：DOP 是误差放大因子，理论上定位误差 $\sigma_r \approx \text{PDOP} \cdot \sigma_{pseudo}$。在仿真中，卫星数量 10-15 颗时 PDOP 约 1.5-2.5，误差与 PDOP 正相关的规律在散点图中得到验证（$r > 0.5$）。

### 项目特点与亮点

1. **伪距修正完整**：Saastamoinen 对流层 + 简化电离层 + 卫星钟差三项修正，RMS 从约 60 m 降至 10 m 以下
2. **代码去重**：module3 的伪距生成函数复用 module1 的 `simulate_pseudorange`，字段名统一为 `iono_error`/`tropo_error`
3. **大样本 ML**：每场景 241 个历元（30s 采样，2h），总样本量约 720，显著优于原来的 26 个
4. **双 GUI**：基础部分和 ML 部分各有独立 GUI，风格一致
5. **DOP 分析可视化**：4 子图分析 DOP 与误差关系，相关系数量化展示

### 不足之处

1. 伪距为模拟生成，与真实观测数据存在理想化差距
2. 电离层使用固定天顶延迟（10 m），未用 Klobuchar 模型动态估算
3. ML 特征未利用历元序列的时间相关性（如卡尔曼滤波）

### 改进建议

1. 引入真实 RINEX OBS 文件，验证真实观测场景下的定位精度
2. 使用 Klobuchar 或广播电离层改正参数进行更精确的电离层修正
3. 引入 XGBoost / LightGBM 或时序模型（LSTM）提升 ML 补偿效果
4. 在 DOP 分析基础上引入加权最小二乘（WLS），以仰角为权重进一步提升精度

## 7. 结束语（心得体会）

通过本次项目课的实践，我对卫星导航定位有了从理论到代码的全面理解。以前只知道 GPS 能定位，现在明白了背后每一步：从星历文件解析、开普勒方程迭代、伪距修正，到最小二乘求解和 DOP 计算，每个环节都有其数学意义和工程挑战。

遇到的最大困难是卫星钟差修正。刚开始以为迭代最小二乘收敛就代表定位成功，但发现 RMS 一直在 60 m 左右，排查了很久才意识到是伪距没有做修正，大量确定性偏差被接收机钟差吸收而无法区分。加入三项修正后，误差立刻下降到 10 m 以内，这让我深刻体会到 GNSS 定位中"修正"的重要性，也理解了为什么实际接收机芯片里有大量的修正算法。

ML 附加题让我看到了数据驱动方法的潜力——在 GNSS 物理模型已经处理了主要误差之后，残余的非线性残差仍然可以被随机森林捕捉到，取得额外 40%+ 的 RMS 改善。这也让我意识到传统模型与 ML 结合是未来 GNSS 精度提升的重要方向。

对北斗系统的理解也加深了：BDS-3 不仅有 MEO 卫星，还有提供亚太区域增强的 GEO 和 IGSO 卫星，它们的轨道特性不同导致需要不同的坐标系修正策略。北斗已经实现了全球覆盖，作为中国自主研发的导航系统，北斗在精度和可用性上已可与 GPS 媲美。
