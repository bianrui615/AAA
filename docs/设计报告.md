# 北斗定位解算全流程软件系统 — 设计报告

## 1. 项目定位

本项目使用 Python 实现北斗三号 BDS-3 CNAV 广播星历下的单点定位（SPP）全流程。当前实现不读取真实 OBS 观测文件，而是以 NAV 广播星历、给定接收机真实坐标和模拟伪距误差模型构造可复现实验数据。

系统由基础部分和提高部分组成：

- `basic/`：NAV 解析、卫星位置与钟差计算、模拟伪距、SPP 解算、连续定位、报告与基础 GUI。
- `enhance/`：机器学习数据集构建、线性回归与随机森林训练、坐标误差补偿、评估图表与提高部分 GUI。

## 3. 基础流程设计

基础流程入口为 `basic/module5.py`，GUI 入口为 `basic/rinex_gui.py`。两者最终都通过 `module5._get_output_dir(nav_file_path)` 按 NAV 文件名生成输出目录，例如：

```text
nav/tarc0910.26b_cnav
→ outputs/basic/tarc0910_26b_cnav/
```

基础流程按以下顺序执行：

1. `module1.parse_nav_file()` 解析 BDS-3 CNAV 广播星历。
2. `module2` 批量计算指定历元的卫星 ECEF 坐标、卫星钟差和伪距修正调试记录。
3. `module3` 生成模拟伪距并使用迭代最小二乘进行单历元 SPP。
4. `module4.run_continuous_positioning()` 对多个历元连续解算并统计误差。
5. `module5` 汇总各模块状态并生成系统测试报告。

基础部分主要输出包括：

## 4. GUI 与三场景设计

`basic/rinex_gui.py` 面向单个 NAV 文件，支持导入文件、设置时间范围、采样间隔、随机种子、收敛阈值、高度角截止阈值、静态或动态接收机轨迹，并实时显示结果表格和图表。

`basic/gui_scenario_runner.py` 面向课程三场景批量评估，输出目录固定为：

```text
outputs/basic/gui_scenario_runner/scenario_1/
outputs/basic/gui_scenario_runner/scenario_2/
outputs/basic/gui_scenario_runner/scenario_3/
```

当前默认三场景为：

| 场景 | NAV 文件 | 时间范围 | 轨迹 |
| --- | --- | --- | --- |
| 场景 1 | `nav/tarc0910.26b_cnav` | 2026-04-01 00:00 至 06:00 | 静态接收机 |
| 场景 2 | `nav/tarc1220.26b_cnav` | 2026-05-02 00:00 至 06:00 | 动态直线运动 |
| 场景 3 | `nav/tarc1230.26b_cnav` | 2026-05-03 00:00 至 06:00 | 动态折线轨迹 |

## 5. 提高部分设计

提高部分入口为 `enhance/run_enhance.py`，也可通过 `enhance/enhance_gui.py` 操作。场景名统一为：

```text
scenario1
scenario2
scenario3
```

数据构建逻辑如下：

1. 优先读取 `outputs/basic/gui_scenario_runner/scenario_1..3/module4_连续定位结果.csv`。
2. 读取成功时，补充伪距统计特征并构建 `outputs/enhance/ml/机器学习数据集.csv`。
3. 若基础三场景结果不存在，则按 `enhance/enhance_config.py` 重新运行并保存到 `outputs/enhance/ml/scenarios/scenario1..3/`。

当前模型包括：

- `LinearRegression`
- `RandomForestRegressor`

训练采用按 `scenario_name` 整场景划分训练集和测试集，避免同一场景相邻历元同时进入训练集与测试集造成数据泄漏。`train_models.py` 支持可选 `GridSearchCV`，默认关闭。

提高部分输出包括：

```text
outputs/enhance/ml/机器学习数据集.csv
outputs/enhance/ml/scenarios/scenario1/module4_连续定位结果.csv
outputs/enhance/ml/scenarios/scenario2/module4_连续定位结果.csv
outputs/enhance/ml/scenarios/scenario3/module4_连续定位结果.csv
outputs/enhance/models/线性回归模型.joblib
outputs/enhance/models/随机森林模型.joblib
outputs/enhance/ml/predictions/线性回归_补偿预测.csv
outputs/enhance/ml/predictions/随机森林_补偿预测.csv
outputs/enhance/ml/模型对比汇总.csv
outputs/enhance/ml/补偿效果统计.txt
outputs/enhance/ml/技术报告.txt
outputs/enhance/figures/*.png
```

## 6. 关键约束

- 当前不读取 `.obs` 文件，所有伪距由模拟误差模型生成。
- 默认仅处理北斗卫星记录。
- 核心 GNSS 计算由项目代码实现，不依赖 RTKLIB、georinex 等 GNSS 专用库。
- 基础输出和提高输出相互隔离：`outputs/basic/` 与 `outputs/enhance/` 分开存放。
- 同一个 NAV 文件通过 `rinex_gui.py` 或 `module5.py` 重复运行时，会复用同一个 `outputs/basic/<nav目录>/`，同名输出文件会被新结果覆盖。
