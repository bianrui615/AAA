# 北斗定位解算全流程软件系统 — 设计报告

---

## 1. 功能需求与技术指标

### 1.1 项目背景与目标

本项目基于北斗三号（BDS-3）广播星历，使用 Python 实现卫星导航单点定位（SPP, Single Point Positioning）全流程软件系统。项目不依赖真实观测文件（OBS），而是通过高斯误差模型模拟伪距，在此基础上进行伪距修正与最小二乘定位解算，最终利用机器学习（ML）方法对残余误差进行补偿，验证数据驱动方式在 GNSS 精度提升中的可行性。

系统的总体目标是：在标准单频单点定位场景下，通过三项确定性伪距修正（卫星钟差、对流层延迟、电离层延迟），将三维定位 RMS 误差从约 60 m 降低到 10 m 以内；在此基础上利用随机森林模型进一步将误差降低 30% 以上。

### 1.2 功能性需求清单

| 编号 | 功能模块 | 具体需求描述 | 优先级 |
|------|---------|------------|--------|
| F-01 | RINEX NAV 文件解析 | 支持 RINEX 3.x CNAV 格式，正确提取 BDS-3 各型卫星（MEO/GEO/IGSO）的广播星历参数，包括轨道根数、钟差系数、健康状态等 | 必须 |
| F-02 | 卫星位置与钟差计算 | 基于开普勒轨道方程、地球引力摄动修正和相对论效应，计算任意历元的卫星 ECEF 坐标和钟差修正量；GEO 卫星须加入 -5° 倾角旋转修正 | 必须 |
| F-03 | 模拟伪距生成 | 使用高斯随机误差模型（含 SISRE、电离层、对流层、接收机钟差、噪声五项）生成模拟伪距，支持随机种子保证可复现 | 必须 |
| F-04 | 伪距确定性修正 | 实现 Saastamoinen 简化对流层模型、基于仰角的电离层映射函数、卫星钟差（含相对论效应）三项修正，修正后伪距作为 SPP 输入 | 必须 |
| F-05 | 单点定位 SPP 解算 | 实现迭代加权最小二乘（当前为等权），收敛阈值 $10^{-4}$ m，最多迭代 12 次，输出 ECEF 坐标、接收机钟差、PDOP、GDOP | 必须 |
| F-06 | 高度角截止过滤 | 支持可配置的高度角截止阈值（默认 0°，可设 5°/10°/15°），低于阈值的卫星剔除出解算 | 必须 |
| F-07 | 连续多历元定位 | 按指定时间间隔（默认 30s）循环解算，输出逐历元定位结果 CSV，统计成功率、RMS 误差、平均 DOP | 必须 |
| F-08 | DOP 与精度关系分析 | 统计计算 PDOP/GDOP 与三维误差的 Pearson 相关系数，生成散点图+拟合直线+箱线图+误差直方图的 2×2 分析图 | 必须 |
| F-09 | 多场景系统测试 | 支持 3 个及以上不同场景（不同位置、NAV 文件、随机种子、高度角设置）的批量自动化测试 | 必须 |
| F-10 | 基础部分 GUI | 基于 PyQt5 的图形界面，支持 NAV 文件导入、参数配置、后台线程解算、实时 matplotlib 图表嵌入展示 | 必须 |
| F-11 | ML 多场景数据集构建 | 自动运行多个场景并收集特征（19 维）和标签（三维坐标误差），输出 ml_dataset.csv | 必须 |
| F-12 | ML 模型训练 | 支持 LinearRegression 和 RandomForestRegressor 两种模型训练，支持 GridSearchCV 超参数调优，保存 joblib 文件 | 必须 |
| F-13 | 误差补偿与评估 | 使用训练好的模型对测试集进行误差预测与坐标补偿，统计 RMS 改善率，生成对比图表 | 必须 |
| F-14 | ML 独立 GUI | 独立于基础部分的 PyQt5 界面，支持数据集构建→模型训练→补偿评估全流程操作，结果嵌入标签页展示 | 必须 |

### 1.3 技术指标要求

**定位精度指标**：

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 修正后三维 RMS 误差 | < 10 m | 施加三项伪距修正后，连续历元统计值 |
| 修正后单历元最大误差 | < 20 m | 95% 置信区间 |
| ML 补偿后 RMS 误差（随机森林） | < 7 m | 相对修正前 RMS 改善率 > 30% |
| 修正前三维 RMS 误差 | 约 60 m | 无修正时接收机钟差全部压入位置偏差 |

**系统性能指标**：

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 单历元 SPP 解算时间 | < 50 ms | 包含星历选取、伪距修正、最小二乘迭代 |
| 连续 241 历元总耗时 | < 30 s | 30s 间隔，2 小时数据 |
| ML 数据集构建（3 场景） | < 120 s | 720 历元总量 |
| ML 随机森林训练时间 | < 120 s | 500+ 样本，200 棵树 |
| SPP 收敛率 | > 95% | 正常卫星可见情况下 |

**可靠性指标**：

| 指标 | 要求 |
|------|------|
| 结果可复现性 | 给定相同随机种子，两次运行结果完全一致（浮点精度内） |
| 异常处理 | 星历缺失、卫星数不足（< 4 颗）、矩阵奇异等情形均有明确错误提示，不崩溃 |
| 跨平台兼容 | Windows 10/11 和 Linux（Ubuntu 20.04+）均可正常运行 |

### 1.4 约束与假设

1. **不读取真实 OBS 文件**：所有伪距由 `module1.simulate_pseudorange` 中的高斯误差模型生成，这是本项目的特殊约束，使实验完全可控可复现。
2. **仅处理 BDS-3 卫星**：导航文件中以字母 `C` 开头的卫星标识（C01-C63），不处理 GPS（G）、GLONASS（R）、Galileo（E）等系统。
3. **单频单点定位**：不进行双频电离层延迟组合消除（L1/L2 组合），电离层修正采用模型修正。
4. **手写 GNSS 算法**：禁止调用 RTKLIB、georinex、pyproj、astropy 等第三方 GNSS 或大地测量库，所有卫星位置计算、坐标转换、DOP 计算均手动实现。
5. **依赖包白名单**：numpy（矩阵运算）、matplotlib（可视化）、scikit-learn（ML 模型）、joblib（模型持久化）、PyQt5（GUI），不超出此范围。
6. **接收机运动模型**：主要场景假设接收机静止；GUI 支持匀速直线运动轨迹作为扩展选项，但验收指标基于静止场景。

## 3. 模块分析

### 3.1 模块一：NAV 解析与伪距生成（module1.py，约 1250 行）

**职责**：本模块是整个系统的数据入口，承担三项核心功能——RINEX 导航文件解析、卫星 ECEF 坐标/钟差计算、模拟伪距生成。所有下游模块（module2-4 及 enhance）均依赖此模块提供的数据结构和计算函数。

**主要函数**：

| 函数名 | 功能 | 关键参数/返回值 |
|--------|------|----------------|
| `parse_nav_file(path)` | 解析 RINEX 3.x CNAV 文件 | 返回 `(nav_data, parse_info)`，其中 nav_data 为 `Dict[str, List[BroadcastEphemeris]]` |
| `select_ephemeris(nav_data, sat_id, epoch_time, healthy_only)` | 为给定历元选择最佳星历 | 选取 $|t - t_{oe}|$ 最小的健康星历 |
| `compute_satellite_position(eph, epoch_time)` | 计算卫星 ECEF 坐标 | 返回 `(x, y, z)`（米），含 GEO 旋转修正 |
| `compute_satellite_clock_bias(eph, epoch_time)` | 计算卫星钟差 | 返回 `(clock_bias_s, relativistic_s)`（秒） |
| `simulate_pseudorange(rho, rng)` | 生成模拟伪距 | 返回包含各误差分项的字典，字段：rho, sisre_error, iono_error, tropo_error, receiver_clock_error, noise_error, pseudorange |
| `compute_elevation(receiver_xyz, satellite_xyz)` | 计算卫星高度角 | 返回度（°），通过 ECEF→ENU 变换实现 |
| `ecef_to_blh(x, y, z)` | ECEF→大地坐标转换 | 返回 `(lat°, lon°, height_m)`，WGS84 椭球，迭代法 |
| `compute_geometric_range(p1, p2)` | 两点几何距离 | 返回米 |

**BroadcastEphemeris 数据结构**（关键字段）：

```python
@dataclass
class BroadcastEphemeris:
    sat_id: str           # 卫星编号（如 "C01"）
    toc: datetime         # 钟参考时刻
    toe: datetime         # 轨道参考时刻
    af0: float            # 钟差常数项（s）
    af1: float            # 钟差速率项（s/s）
    af2: float            # 钟差加速度项（s/s²）
    sqrt_a: float         # 半长轴平方根（m^0.5）
    e: float              # 轨道偏心率
    i0: float             # 轨道倾角（rad）
    OMEGA0: float         # 升交点赤经（rad）
    omega: float          # 近地点角距（rad）
    M0: float             # 平近点角（rad）
    delta_n: float        # 平均运动角速度修正量（rad/s）
    i_dot: float          # 轨道倾角变化率（rad/s）
    OMEGA_dot: float      # 升交点赤经变化率（rad/s）
    Crc, Crs: float       # 轨道半径谐波修正（m）
    Cuc, Cus: float       # 升交角距谐波修正（rad）
    Cic, Cis: float       # 轨道倾角谐波修正（rad）
    health: float         # 健康状态（0=健康）
    week_number: int      # BDS 周计数
```

**设计决策**：`simulate_pseudorange` 作为唯一的误差模型实现，module3 的 `generate_simulated_pseudorange_record` 是其薄包装层，这样保证误差模型参数只有一处定义，修改时不会产生不一致。

### 3.2 模块二：卫星位置与钟差输出层（module2.py，约 380 行）

**职责**：调用模块一的计算函数，对某个指定历元的全部可见卫星批量计算位置与钟差，格式化输出到 CSV 和终端，主要服务于单步验证和教学展示。

**主要函数**：

| 函数名 | 功能 |
|--------|------|
| `compute_all_satellite_positions(nav_data, epoch_time, healthy_only)` | 批量计算所有健康卫星的 ECEF 坐标 |
| `save_satellite_positions_to_csv(positions, path)` | 保存卫星坐标到 CSV |
| `print_satellite_summary(positions, clock_biases)` | 终端格式化打印摘要 |

本模块对系统主流程没有强依赖，主要用于单独验证模块一的计算结果。

### 3.3 模块三：SPP 解算与伪距修正（module3.py，约 680 行）

**职责**：本模块是定位解算的核心，包含两大功能：（1）三项伪距确定性修正函数；（2）迭代最小二乘 SPP 解算器。

**伪距修正函数**：

| 函数名 | 功能 | 数值示例 |
|--------|------|---------|
| `saastamoinen_tropospheric_delay(elevation_deg, ...)` | Saastamoinen 对流层延迟（简化版） | 90°≈2.3 m，30°≈4.6 m |
| `simple_ionospheric_delay(elevation_deg, zenith_delay_m=10.0)` | 余割映射电离层延迟 | 90°=10.0 m，30°=20.0 m |
| `apply_pseudorange_corrections(raw_pr, clock_bias_s, elevation_deg, ...)` | 综合三项修正 | 返回 (corrected_pr, 修正量明细字典) |

**SppSolution 数据结构**：

```python
@dataclass
class SppSolution:
    status: str           # "成功"或"失败"
    converged: bool       # 是否在 max_iter 内收敛
    x, y, z: float        # 解算得到的 ECEF 坐标（米）
    lat, lon: float       # 大地纬度/经度（度）
    height: float         # 大地高程（米）
    clock_bias: float     # 接收机钟差估计值（米）
    pdop: float           # 位置精度衰减因子
    gdop: float           # 几何精度衰减因子
    iterations: int       # 实际迭代次数
```

## 4. 关键技术

### 4.1 BDS-3 广播星历卫星位置计算

卫星位置计算是整个系统最核心的算法，直接影响定位精度。计算流程严格按照 BDS-3 ICD 文档：

**第一步：计算修正后的平均运动角速度**

$$n_0 = \sqrt{\frac{\mu}{a^3}}, \quad n = n_0 + \Delta n$$

其中 $\mu = 3.986004418 \times 10^{14}\,\text{m}^3/\text{s}^2$（北斗系统地球引力常数），$a = (\sqrt{a})^2$ 为轨道长半轴。

**第二步：计算历元时差并归一化**

$$t_k = t - t_{oe}$$

当 $|t_k| > 302400\,\text{s}$（半周）时，加减 604800 s 修正，避免跨周边界错误。

**第三步：计算平近点角和偏近点角**

$$M_k = M_0 + n \cdot t_k$$

开普勒方程迭代求解偏近点角 $E_k$（通常 8-12 次迭代收敛到机器精度）：

$$E_{k+1} = M_k + e \sin E_k$$

**第四步：计算真近点角和升交角距**

$$\nu_k = \text{atan2}\!\left(\frac{\sqrt{1-e^2}\sin E_k}{1 - e\cos E_k},\; \frac{\cos E_k - e}{1-e\cos E_k}\right)$$

$$u_k = \omega + \nu_k$$

**第五步：二阶谐波摄动修正**

$$\delta u = C_{us}\sin 2u_k + C_{uc}\cos 2u_k$$
$$\delta r = C_{rs}\sin 2u_k + C_{rc}\cos 2u_k$$
$$\delta i = C_{is}\sin 2u_k + C_{ic}\cos 2u_k$$

**第六步：轨道面内坐标 → 地固系 ECEF**

计算修正后的半径 $r_k$、倾角 $i_k$，在轨道面内得到坐标 $(x', y')$，再通过旋转矩阵转到 ECEF：

$$\Omega_k = \Omega_0 + (\dot{\Omega} - \omega_e) t_k - \omega_e t_{oe}$$

其中 $\omega_e = 7.2921150 \times 10^{-5}\,\text{rad/s}$ 为地球自转角速度。

**第七步：GEO 卫星特殊处理**

GEO 卫星（C01-C05, C59-C63）的轨道倾角接近 0°，北斗坐标系与 CGCS2000 之间存在约 $-5°$ 的倾角偏差，需要在地固系旋转之外额外叠加绕 X 轴旋转 $-5°$ 的矩阵：

$$\mathbf{R}_x(-5°) = \begin{pmatrix} 1 & 0 & 0 \\ 0 & \cos(-5°) & -\sin(-5°) \\ 0 & \sin(-5°) & \cos(-5°) \end{pmatrix}$$

### 4.2 伪距确定性误差修正

**修正必要性分析**：

模拟伪距包含五项误差，其中：
- **接收机钟差**（约 60 m）：以公共偏差形式出现在所有卫星，由最小二乘的第 4 个参数（clock_bias）吸收，不影响位置精度
- **卫星钟差**（约 40-80 m 量级）：各卫星不同，无法被接收机钟差吸收，**必须在解算前修正**
- **电离层延迟**（约 10-20 m）：各卫星不同（高度角依赖），**必须修正**
- **对流层延迟**（约 2-5 m）：各卫星不同，**须修正以满足 10 m 以内精度要求**
- **SISRE 和噪声**（合计约 0.6 m）：随机误差，无法确定性修正，残留在残差中

**Saastamoinen 对流层修正推导**：

Hopfield 模型将大气分为干分量（主导）和湿分量，Saastamoinen 简化形式为：

$$T_{tropo}(\theta) = \frac{0.002277}{\sin\theta}\left[P_0 + \left(\frac{1255}{T_0}+0.05\right)e_v - B \tan^2 z\right]$$

其中 $B \approx \tan^2 z$（$z$ 为天顶角），标准大气参数 $P_0=1013.25\,\text{hPa}$，$T_0=288.15\,\text{K}$，湿度 50% 时 $e_v \approx 3.1\,\text{hPa}$。天顶方向修正量约 2.3 m，仰角 30° 时约 4.6 m（2 倍关系）。

**简化电离层修正推导**：

单层电离层（SLM）假设电离层集中在 350 km 高度，垂直方向延迟为 $I_z$，倾斜方向通过映射函数：

$$T_{iono}(\theta) = \frac{I_z}{\sin\theta}$$

本项目取 $I_z = 10\,\text{m}$（与伪距模拟的电离层均值一致），能消除电离层的系统性偏差，但无法消除 $\mathcal{N}(0, 3)$ 的随机涨落部分（残余约 3-6 m，高度角依赖）。

**修正公式综合**：

$$P_{corr} = P_{raw} + c \cdot \delta t_{sv} - T_{iono}(\theta) - T_{tropo}(\theta)$$

各修正量的符号含义：卫星钟差为正修正（"加回"钟差产生的偏差），电离层和对流层为负修正（"减去"信号传播路径上的额外延迟）。

### 4.3 迭代最小二乘 SPP

**观测方程线性化**：

设接收机当前概略位置为 $(x_0, y_0, z_0)$，概略钟差为 $b_0$，对第 $i$ 颗卫星的伪距观测方程在概略值处泰勒展开：

$$P_i^{corr} = \rho_i + b + \epsilon_i$$

$$\rho_i \approx \rho_i^0 + \frac{\partial\rho_i}{\partial x}\delta x + \frac{\partial\rho_i}{\partial y}\delta y + \frac{\partial\rho_i}{\partial z}\delta z$$

其中方向余弦：

$$\frac{\partial\rho_i}{\partial x} = -\frac{x_s^i - x_0}{\rho_i^0}, \quad \frac{\partial\rho_i}{\partial y} = -\frac{y_s^i - y_0}{\rho_i^0}, \quad \frac{\partial\rho_i}{\partial z} = -\frac{z_s^i - z_0}{\rho_i^0}$$

**设计矩阵与法方程**：

$$\mathbf{H} = \begin{pmatrix} -l_1 & -m_1 & -n_1 & 1 \\ -l_2 & -m_2 & -n_2 & 1 \\ \vdots & & & \vdots \\ -l_n & -m_n & -n_n & 1 \end{pmatrix}, \quad \mathbf{v} = \begin{pmatrix} P_1^{corr} - \rho_1^0 - b_0 \\ \vdots \\ P_n^{corr} - \rho_n^0 - b_0 \end{pmatrix}$$

最小二乘解：$\delta\hat{\mathbf{x}} = (\mathbf{H}^T\mathbf{H})^{-1}\mathbf{H}^T\mathbf{v}$

收敛后更新状态，直到 $\|\delta_{xyz}\| < 10^{-4}\,\text{m}$ 或达到最大迭代次数。

**DOP 矩阵**：

协因数矩阵 $Q = (\mathbf{H}^T\mathbf{H})^{-1}$，其对角元素反映不同方向的精度：

$$\text{PDOP} = \sqrt{Q_{11}+Q_{22}+Q_{33}}, \quad \text{TDOP} = \sqrt{Q_{44}}, \quad \text{GDOP} = \sqrt{\text{PDOP}^2 + \text{TDOP}^2}$$

### 4.4 ECEF ↔ BLH 坐标变换

**BLH → ECEF**（正变换，解析公式）：

$$x = (N + h)\cos\phi\cos\lambda$$
$$y = (N + h)\cos\phi\sin\lambda$$
$$z = (N(1-e^2) + h)\sin\phi$$

其中 $N = a/\sqrt{1-e^2\sin^2\phi}$ 为卯酉圈曲率半径，$a = 6378137.0\,\text{m}$，$e^2 = 0.00669437999014$（WGS84）。

**ECEF → BLH**（逆变换，Bowring 迭代法）：

$$\lambda = \text{atan2}(y, x)$$

迭代直到纬度收敛：

$$\phi_{k+1} = \text{atan2}\!\left(z + e^2 N(\phi_k)\sin\phi_k,\; p\right), \quad p = \sqrt{x^2+y^2}$$

高程：$h = p\cos^{-1}\phi - N(\phi)$（当 $|\phi| < 45°$ 时用此式），或 $h = z\sin^{-1}\phi - N(1-e^2)$。

### 4.5 PyQt5 多线程 GUI 设计

GUI 的核心挑战是防止长时间定位任务阻塞主 UI 线程（Qt 所有界面操作必须在主线程）。解决方案是将计算密集型操作封装在 `QThread` 子类中，通过 `pyqtSignal` 跨线程通信：

```
主线程（UI）                        工作线程（QThread）
    │                                     │
    │── 用户点击"运行" ─────────────→     │── run() 执行 SPP/ML 计算
    │                                     │
    │←── progress_signal(value) ─────────│── 每历元发出进度
    │    更新进度条                        │
    │←── log_signal(text) ──────────────│── 发出日志文本
    │    追加到日志框                      │
    │←── finished_signal(result) ────────│── 计算完成，发出结果字典
    │    更新表格/图表                     │
```

`stdout` 重定向技术：将 `sys.stdout` 替换为自定义的 `_LogRedirect` 对象，其 `write` 方法通过 `log_signal` 将打印内容转发到 GUI 日志框，使所有 `print` 语句输出自动显示在界面中。

### 4.6 机器学习误差补偿流程

ML 误差补偿的核心思路：用已知真实坐标的历史定位数据训练一个从"定位特征"到"定位误差"的映射函数，然后对新历元的 SPP 结果进行误差预测并补偿。

**为什么选这 19 个特征？**

1. **PDOP/GDOP**：衡量卫星几何结构，理论上 $\sigma_{position} \approx \text{PDOP} \times \sigma_{pseudorange}$，与误差高度相关
2. **伪距统计量（均值/标准差）**：均值反映平均观测距离，标准差反映各卫星观测条件的均匀性
3. **各误差均值**：在测试中虽然伪距已做了确定性修正，但每颗卫星的实际误差是高斯随机量，均值特征能部分反映整体误差水平的随机实现
4. **高度角统计**：低仰角卫星带入更大的对流层/电离层残余误差，最低仰角和平均仰角是该因素的代理指标
5. **迭代次数**：迭代次数多的历元通常说明初值较差或卫星几何复杂，与误差存在间接关联

**补偿精度期望**：

| 方法 | 理论 RMS 改善 | 原因 |
|------|------------|------|
| 线性回归 | 15-30% | 只能捕捉误差与特征之间的线性关系，而实际误差与 PDOP 的关系是近似线性的 |
| 随机森林 | 30-50% | 能捕捉高度角与误差的非线性映射（低仰角误差呈指数增长），通过集成学习降低方差 |

## 6. 优化方向

### 6.1 定位算法精度优化

**加权最小二乘（WLS）**：当前实现为等权最小二乘，对所有卫星赋予相同权重。实际上，高仰角卫星的对流层/电离层残余误差更小，应赋予更高权重。改进方法：引入以仰角为基础的观测权矩阵：

$$\mathbf{W} = \text{diag}\left(w_1, w_2, \ldots, w_n\right), \quad w_i = \sin^2\theta_i$$

加权法方程变为 $\delta = (\mathbf{H}^T\mathbf{W}\mathbf{H})^{-1}\mathbf{H}^T\mathbf{W}\mathbf{v}$，预期可将 RMS 误差额外降低 10-20%。

**Klobuchar 电离层模型**：当前电离层修正使用固定天顶延迟 10 m，更精确的方法是使用 BDS-3 广播的 Klobuchar 系数（存储在 NAV 文件头部），根据接收机位置、太阳时角和卫星方位角动态估算电离层延迟，可将电离层残余误差从约 3-6 m 降低到 1-2 m。

**精密对流层模型（GPT3）**：当前 Saastamoinen 简化版使用标准大气参数（固定气压/温度），若引入 GPT3（全球气压温度模型），使用历元时间和接收机位置查表得到实时气压/温度，可提高对流层修正精度约 0.5-1 m。

**粗差剔除优化**：当前粗差剔除（`enable_pseudorange_outlier_filter`）默认关闭，因为模拟数据没有真实粗差。若扩展到真实观测数据，建议启用 MAD（中位数绝对偏差）方法进行稳健估计，阈值取 3σ。

### 6.2 机器学习模型优化

**XGBoost / LightGBM**：相比随机森林，梯度提升树（Gradient Boosting Trees）通常在表格数据上取得更好的性能，且 LightGBM 的训练速度比随机森林快 5-10 倍。可以作为第三个模型加入比较。

**时序特征利用**：当前特征仅使用单历元信息，不包含历史信息。实际上相邻历元的误差存在时间自相关性（前一时刻的误差对预测当前时刻有帮助）。改进方向：使用滑动窗口特征（前 n 个历元的误差均值、趋势等），或引入 LSTM/GRU 等时序模型。

**在线学习**：当前为离线批量训练，部署后模型固定不更新。可以引入在线增量学习（如 `sklearn.linear_model.SGDRegressor`），使模型随新数据持续更新，应对不同时间段的系统偏差变化。

**神经网络**：对于大样本场景（> 10000 个历元），全连接神经网络（MLP）可以捕捉更复杂的非线性关系。建议先用较浅的网络（2-3 层，每层 64-128 节点）作为基线，避免过拟合。

**特征工程扩展**：
- 添加历元时间的周期编码（sin/cos 变换，反映一天内的电离层周期变化）
- 添加接收机纬度/高度（影响对流层延迟的基准值）
- 添加 VDOP（垂直精度衰减因子，比 PDOP 更能反映高程方向的误差）

### 6.3 系统性能优化

**并行化历元计算**：当前连续定位按历元串行执行，每历元的卫星位置计算和 SPP 解算相互独立。使用 `multiprocessing.Pool` 或 `concurrent.futures.ProcessPoolExecutor` 并行处理，理论上可线性加速（取决于 CPU 核心数）。对于 720 历元的 ML 数据集构建，预期加速比为 4-8 倍（4-8 核 CPU）。

**星历选取缓存**：`select_ephemeris` 每次都遍历该卫星的所有星历记录寻找最近者。对于 2h/30s 场景，同一颗卫星会重复选同一条星历约 100 次。引入 LRU 缓存（`functools.lru_cache`）可将重复查询时间降为 O(1)。

**numpy 向量化**：当前卫星位置计算使用逐颗循环，可以将多颗卫星的轨道传播公式向量化（将 $n$ 颗卫星的标量参数打包为 1D 数组，同时计算），减少 Python 循环开销。

### 6.4 功能扩展优化

**真实观测数据支持**：当前系统不读取 OBS 文件，这是课程要求的约束。若解除此约束，可以引入真实 GNSS 观测数据进行真实定位实验，进一步验证算法在非理想条件下的表现。

**多系统 GNSS 融合**：在 NAV 解析层面增加对 GPS（G）、GLONASS（R）、Galileo（E）卫星的支持，在 SPP 层面引入系统间偏差（ISB）参数，实现多系统组合定位，预期将可见卫星数提升到 20-30 颗，PDOP 降至 1.0-1.5，定位精度进一步提高。

**卡尔曼滤波平滑**：当前各历元独立解算，存在历元间跳变。引入扩展卡尔曼滤波（EKF）将历元之间的运动模型（匀速直线、CV 模型等）作为先验，结合 SPP 观测量进行融合，可以在卫星数量不足时仍保持较好的定位连续性。

**差分定位（DGNSS）扩展**：若引入参考站（其真实坐标已知），可以计算参考站伪距残差（差分修正量），通过差分消除大部分空间相关误差（对流层、电离层的公共部分），将定位精度提高到 1-3 m 量级。

### 6.5 GUI 与用户体验优化

**地图底图集成**：当前轨迹图使用纯白色背景的经纬度平面图，可引入 `folium` 或在线地图 tile（注意：需联网）生成带有地理背景的定位轨迹图，增强可读性。

**实时回放功能**：在 GUI 中增加"回放"模式，按历元时间依次显示卫星位置和接收机位置，以动画形式展示定位过程，增强教学效果。

**配置文件持久化**：当前参数（NAV 路径、时间范围、随机种子等）每次重启后复位。可以将用户配置保存为 JSON 文件，下次启动时自动加载，提高使用便捷性。

**批量导出报告**：一键将所有输出图表和统计文本打包为 ZIP 文件，便于提交和分享实验结果。
